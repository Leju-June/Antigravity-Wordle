# Wordle Clone 기획안 (Reflex Framework)

## 1. 프로젝트 개요
Python 기반의 풀스택 웹 프레임워크인 **Reflex**를 사용하여 인기 단어 퍼즐 게임인 'Wordle'의 클론 앱을 개발합니다. 파이썬만으로 프론트엔드와 백엔드 상태를 모두 관리하는 Reflex의 장점을 활용합니다.

## 2. 주요 기능 및 요구사항

### 2.1. 게임 플레이 로직
- **정답 단어 선정**: 5글자로 된 영단어 리스트 중 무작위로 오늘의 단어를 선정합니다.
- **기회 제한**: 플레이어에게 총 6번의 단어 입력 기회가 주어집니다.
- **단어 입력**: 
  - 키보드 클릭 또는 실제 키보드 입력을 통해 5글자 단어를 제출할 수 있습니다.
  - 제출 시 사전에 있는 단어인지 검증합니다.

### 2.2. 단어 체크 및 피드백 시각화
단어를 제출하면 각 알파벳과 위치에 따라 3가지 색상으로 피드백을 제공합니다.
- 🟩 **초록색 (Correct)**: 알파벳이 정답에 포함되어 있고, 위치도 정확함.
- 🟨 **노란색 (Present)**: 알파벳이 정답에 포함되어 있으나, 위치가 다름.
- ⬜ **회색 (Absent)**: 알파벳이 정답에 존재하지 않음.

### 2.3. 승패 판정
- **승리**: 6번의 기회 안에 정답과 완전히 일치하는 단어를 제출함.
- **패배**: 6번의 기회를 모두 소진하였으나 정답을 맞추지 못함. 팝업이나 화면을 통해 정답을 알려줍니다.

### 2.4. 결과 공유 기능
- 게임 종료 후 자신의 결과를 클립보드에 복사할 수 있는 '공유하기' 기능을 제공합니다.
- **공유 포맷 예시**:
  ```
  Wordle Clone 3/6
  
  🟩🟩⬜⬜🟨
  🟩🟩🟩⬜⬜
  🟩🟩🟩🟩🟩
  ```

## 3. 프로젝트 구조 (예상)

```text
wordle_clone/
├── .context/
│   └── proposal.md         # 프로젝트 기획안 (현재 파일)
├── assets/                 # 이미지, 폰트 등 정적 리소스
├── wordle_clone/
│   ├── __init__.py
│   ├── wordle_clone.py     # 메인 뷰 및 라우팅
│   ├── state.py            # 게임 상태(State), 로직 및 이벤트 핸들러
│   ├── components.py       # 재사용 가능한 UI 컴포넌트 (키보드, 그리드 등)
│   └── words.py            # 정답 단어 및 허용 단어 데이터베이스
└── rxconfig.py             # Reflex 설정 파일
```

## 4. UI/UX 디자인 (Aesthetics)
- **심플하고 직관적인 레이아웃**: 웹뷰나 모바일 화면 모두에 대응하는 반응형 디자인 설계.
- **다크 모드 지원**: 기본적으로 어두운 테마 기반이거나 시스템 설정에 따라 토글되도록 UI 컬러 배스킷 구성.
- **부드러운 애니메이션**: 알파벳이 입력될 때의 팝 효과와 정답 확인 시 개별 칸이 뒤집히는 타일 3D 회전 애니메이션 구현 (Reflex 컴포넌트 및 CSS 활용).
- **키보드 레이아웃**: 입력된 알파벳의 상태(정답, 부분 정답, 오답)를 가상 키보드 UI에도 동일한 색상으로 반영하여 플레이어의 편의성 극대화.

## 5. 단계별 구현 계획
1. **프로젝트 초기화 및 기본 구조 세팅**: Reflex 프로젝트 생성 및 디렉터리 분리.
2. **단어 데이터 로드**: 단어장 리스트(words.py) 구성 및 정답 추출 로직 작성.
3. **상태(State) 정의**: 게임의 현재 상태(입력된 단어들, 현재 시도 횟수, 게임 종료 여부 등)를 관리하는 State 클래스 구현.
4. **글자 판별 로직 구현**: 제출된 단어를 정답과 비교해 초록, 노란색, 회색 상태를 계산.
5. **UI 구현**: 입력 그리드(5x6) 및 가상 키보드 컴포넌트를 만들고 State와 연결.
6. **결과 공유 및 모달 팝업 추가**: 게임 종료 시 알림과 결과 문자열 복사 로직 구현.
